«Відключені» кнопки, створені з елемента <a>, ведуть себе трохи інакше:

  • Елемент <a> не підтримує атрибут disabled, так що для відповідних властивостей замість нього треба додавати клас .disabled;
  • Деякі future-friendly стилі, які підтримуються не всіма юзер-агентами, (тобто «можливо будуть підтримуватися в майбутньому») включені в <a> для відключення всіх подій pointer-events на кнопках-якорях. У браузерах, які підтримують цю властивість, ви зовсім не побачите деактивований курсор.
  • Відключені кнопки повинні містити атрибут aria-disabled="true" для вказівки допоміжним технологіям стану елемента.

Попередження про функціональність посилань

Клас .disabled використовує події-покажчики pointer-events: none, щоб спробувати вимкнути функціональність посилань для <a>, але ця властивість CSS ще не повністю стандартизовано. Крім того, навіть у браузерах, які підтримують властивість події-покажчика pointer-events: none, при навігації з клавіатури користувачі клавіатури для слабозорих та інших допоміжних технологій можуть ненароком пройти за цим посиланням. Так що для повної «безпеки», на додаток до aria-disabled="true" додайте в ці посилання атрибут tabindex="-1" для запобігання можливості «знаходження» їх з клавіатури, та використовуйте власний JavaScript, щоб повністю вимкнути їхню функціональність.

Про картки

Картка - це гнучкий контейнер, що розширюється. Вона включає опції заголовків та футерів, широкий спектр контенту, контекстуальні кольори бекграунду та потужні опції відображення. Якщо ви знайомі з Bootstrap 3, картки замінюють наші старі панелі, заглиблення та ескізи. Функціональність, аналогічна цим компонентам, доступна як класів-модифікаторів для карток.

Приклад

Картки створені із застосуванням якомога меншої кількості коду та стилів, але вони все ж таки мають безліч інструментів для налаштування та контролю. Їх легко вирівнювати і поєднувати коїться з іншими компонентами Bootstrap, т.к. вони створені за допомогою "флексбоксу". За замовчуванням вони не мають margin, тому за необхідності використовуйте утиліти відступів.

Внизу – приклад базової картки фіксованої ширини зі змішаним вмістом. Картки не мають фіксованої ширини за умовчанням, тому вони будуть автоматично заповнювати повну ширину батьківського елемента. Цей параметр можна налаштувати за допомогою різних опцій розмірів.

Карусель

Компонент слайд-шоу для циклічного повторення елементів -

Як це працює

Карусель - це слайд-шоу для відображення серії вмісту, створена на основі CSS 3D-трансформацій і трохи на JavaScript. Вона працює з текстом, зображеннями або звичайною розміткою. Вона також підтримує кнопки “next/prev”.

У браузерах з підтримкою API видимості сторінок< елементи каруселі не будуть змінюватися, коли сторінка невидима користувачеві (наприклад, коли вкладка браузера неактивна, або вікно браузера мінімізовано - кнопка "квадратик" зліва від хрестика "закрити браузер", і т.д.).

Ефект анімації цього компонента залежить від медіа-запиту prefers-reduced-motion. Дивіться розділ редукції анімацій.

Майте на увазі, що вкладені каруселі не підтримуються, а каруселі в цілому не дуже бажані стандартам доступності.

Приклад

Клас .active додають до одного зі слайдів, інакше карусель не буде видно. Також для впровадження елементів керування необхідно встановити унікальний ID елементу класу .carousel, особливо, якщо у вас багато каруселів на одній сторінці. Елементи управління та індикатори повинні мати атрибут data-target (або href для <a>), який збігається з ID елемента класу .carousel.

Задайте унікальний id класу .carousel для можливості гнучкого управління, особливо якщо ви використовуєте багато каруселів на сторінці.

Згортаний контент

Переключайте видимість контенту у вашому проекті за допомогою декількох класів та наших плагінів JavaScript.

Як це працює

Плагін згортання JavaScript використовується для показу приховування контенту. Кнопки або посилання використовуються як тригери, націлені на певні елементи, які треба приховати. Зміна стану елемента відбувається за допомогою height - зміною його з поточного до 0. У зв'язку з тим, як CSS обробляє анімації, ми не можемо використовувати padding в елементі класу .collapse. Замість padding використовуйте незалежний елемент, що обертає.

Ефект анімації цього компонента залежить від медіа-запиту з обмеженим рухом prefers-reduced-motion. Дивіться розділ спеціальних можливостей нашої документації з редукції анімації.

Приклад

Клікніть по кнопці внизу, щоб показати та приховати елемент за допомогою зміни його класу:

  • .collapse приховує вміст
  • .collapsing застосовується під час переходів
  • .collapse.show показує вміст

Як правило, ми рекомендуємо використовувати кнопку з атрибутом data-bs-target. Хоча це не рекомендується з семантичної точки зору, Ви також можете використовувати посилання з атрибутом href (та role="button"). В обох випадках потрібно data-bs-toggle="collapse".

Деякий заповнювач для компонента згортання. За замовчуванням ця панель прихована, але відкривається, коли користувач активує відповідний тригер. 
<p>

Випадаючі елементи

Переключайте контекстні оверлеї для відображення списків посилань та багато іншого за допомогою плагіна розкривного списку Bootstrap.

Огляд

Випадаючі списки - це контекстні накладання, що перемикаються, для відображення списків посилань і багато іншого. Вони зроблені інтерактивними за допомогою увімкненого плагіна JavaScript для меню Bootstrap. Функціональність toggle у випадаючих елементах запускається по кліку, а не за наведенням - це навмисне дизайнерське рішення.

Випадаючі елементи «побудовані» на сторонній бібліотеці Popper.js, яка забезпечує динамічне позиціонування та визначення розміру вікна перегляду. Обов'язково увімкніть popper.min.js перед JavaScript Bootstrap або використовуйте bootstrap.bundle.min.js / bootstrap.bundle.js, який містить Popper.js.

Доступність

Стандарт WAI ARIA описує та визначає віджет role="menu" як справжній віджет, але лише для меню «а-ля додаток», що запускає дії або функції. Меню ARIA можуть містити лише пункти меню, чекбокси, "радіо-кнопки", групи "радіо-кнопок" та підменю.

Випадають елементи Bootstrap, з іншого боку, спроектовані для вирішення безлічі завдань і можуть працювати в різних структурах розмітки. Наприклад, можна створити випадаючі елементи, що містять додаткові поля введення та елементи контролю форм, такі як пошук або поле введення логіну. З цієї причини BS4 не «очікує» (і не додає автоматично) жоден з атрибутів (role та aria-), необхідних для меню згідно зі стандартом ARIA.

Однак Bootstrap завжди додає вбудовану підтримку для більшості стандартних взаємодій меню та клавіатури, таких як можливість рухатися через окремі елементи класу .dropdown-item кнопками курсору на клавіатурі та закривати меню кнопкою ESC.

Модальне вікно

Використовуйте модальний JavaScript плагін Bootstrap для додавання діалогових вікон на ваш сайт для лайтбоксів, повідомлень користувачів або контенту, що повністю настроюється.

Як це працює

До того як розпочати роботу з модальними компонентами Bootstrap, ознайомтеся з наступною інформацією, т.к. параметри нещодавно змінилися.

  • Модальні вікна зроблені на HTML, CSS та JavaScript. Вони розміщуються поверх всього документа і блокують прокручування тіла документа <body>, замість нього прокручуючи модальні вікна
  • Клік поза модальним елементом автоматично закриває його.
  • Bootstrap дозволяє активувати лише один модальний елемент за один раз. Вкладені модальні елементи не підтримуються, т.к. ми вважаємо, що вони принесли б незручності.
  • Модальні елементи мають position: fixed, що може викликати іноді їхнє часткове малювання. Ми рекомендуємо розміщувати модальні елементи на самому верху сторінки, коли можливо, щоб уникнути збоїв та впливу інших елементів на коректну роботу модальних елементів .modal.
  • Знову ж таки, через position: fixed є деякі складнощі при використанні їх на мобільних пристроях. Дивись документацію підтримки у браузерах.
  • Через семантику HTML5 атрибут < code>autofocus не працює з модальними елементами Bootstrap. Щоб досягти такого ж ефекту, використовуйте власний JavaScript:
var myModal = document.getElementById('myModal')
                              var myInput = document.getElementById('myInput')
                              
                              myModal.addEventListener('shown.bs.modal', function () {
                                myInput.focus()
                              })
Ефект анімації цього компонента залежить від медіа-запиту з обмеженим рухом prefers-reduced-motion. Дивіться розділ спеціальних можливостей нашої документації з редукції анімації.

Основи

Навігація в Bootstrap має загальну для подібного типу елементів розмітку, від базового класу .nav до активних та «вимкнених» станів. Замінюйте класи-модифікатори для перемикання між стилями.

Базовий компонент .nav створений на флексбоксі, що забезпечує хорошу базу для створення всіх типів навігаційних компонентів. До нього входять кілька стилів, які «перебивають» інші (для роботи зі списками), додані паддинги посилань для збільшення «зони кліку», та базові стилі «вимкнених» станів.

Навігаційна панель

Документація та приклади адаптивної панелі навігації. Включає підтримку брендингу, навігації за допомогою нашого плагіна для згортання.

Як це працює

Ось що вам потрібно знати перед використанням навбару:

  • Навбари вимагають обгортки з класів .navbar і .navbar-expand{-sm|-md|-lg|-xl|-xxl} для чуйності при «складання» і класи, а також класи колірних схем.
  • Навбари та їх вміст за замовчуванням гнучкі. Використовуйте опціональні контейнери для обмеження їхньої горизонтальної ширини.
  • Використовуйте наші класи відступів та «флекс» для контролю над простором та вирівнюванням усередині навбарів.
  • Навбари є чуйними за замовчуванням, але ви можете легко змінити це. Чуйна поведінка залежить від нашого «плагіна згортання» JavaScript.
  • Надайте їм доступність використанням елемента <nav>, або, якщо використовується менш специфічний елемент – наприклад <div>: додайте role="navigation" у кожен набар для надання йому більшої доступності для користувачів допоміжних технологій.
  • Вкажіть поточний елемент за допомогою aria-current="page" для поточної сторінки або aria-current="true" для поточного елемента в наборі.

Ефект анімації цього компонента залежить від медіазапиту prefers-reduced-motion

Підтримуваний контент

У навбарах є вбудована підтримка багатьох суб-компонентів. Вибирайте потрібний:

  • .navbar-brand для назви вашої компанії, продукту або імені проекту.
  • .navbar-nav для навігації повної висоти (включаючи випадаючі елементи).
  • .navbar-toggler для використання з нашим JS-«плагіном згортання» та інших змін навігації.
  • .form-inline для будь-яких органів контролю форм та дій з ними.
  • .navbar-text для додавання вертикально центрованих рядків тексту.
  • .collapse.navbar-collapse для групування та приховування вмісту навбару на певному брейкпойнті батька.
  • Додайте необов'язковий клас .navbar-scroll, щоб встановити максимальну висоту max-height та прокрутити розширений вміст навігаційної панелі.

Поза полотном

Вбудовуйте у свій проект приховані бічні навігаційні панелі, кошики для покупок та багато іншого за допомогою декількох класів та нашого плагіна JavaScript.

Як це працює

Offcanvas - це компонент бічної панелі, який можна переключити за допомогою JavaScript, щоб він відображався з лівого, правого або нижнього країв області перегляду. Кнопки або якорі використовуються як тригери, які прикріплюються до певних елементів, які ви перемикаєте, а атрибути data використовуються для виклику JavaScript.

  • Offcanvas використовує той же код JavaScript, що й модальні вікна. Концептуально вони дуже схожі, але є окремими плагінами.
  • Так само деякі змінні джерело Sass для стилів і розмірів offcanvas успадковуються від змінних модального вікна.
  • При відображенні offcanvas включає стандартний фон, на який можна клацнути, щоб приховати offcanvas.
  • Як і в модальних вікнах, одночасно може відображатися лише одне неактивне полотно.

Увага! Враховуючи, як CSS обробляє анімацію, Ви не можете використовувати margin або translate в елементі .offcanvas. Натомість використовуйте клас як незалежний елемент оболонки.

Ефект анімації цього компонента залежить від медіазапиту prefers-reduced-motion.

Огляд

Прогресбар

Документація та приклади використання налаштованих індикаторів виконання Bootstrap за допомогою складених смуг, анімованих фонів та текстових міток.

Як це влаштовано

Компоненти прогресу створені з двох елементів HTML, CSS для завдання ширини і декількох атрибутів. Ми не використовуємоелемент HTML5 <progress>, що забезпечує можливість вертикального розташування прогрессбарів, можливість їхньої анімації та розміщення текстових лейблів над ними.

  • Ми використовуємо клас .progress як обгортку для індикації максимального значення прогресу.
  • Ми використовуємо внутрішній клас .progress-bar для індикації прогресу.
  • Клас .progress-bar вимагає оформлення себе як рядкового елемента, звичайного класу або CSS для завдання своєї ширини.
  • Клас .progress-bar також вимагає атрибут role та aria, щоб стати відкритими для взаємодії з допоміжними технологіями.

Все це втілено в прикладах нижче.

Bootstrap має багато утиліт для завдання ширини. Залежно від ваших потреб вони можуть допомогти і з швидким налаштуванням прогресу.

                              </div>

Спінери

Вкажіть стан завантаження компонента або сторінки за допомогою спіннерів Bootstrap, створених повністю з використанням HTML, CSS та без JavaScript.

Огляд

"Спінери" Bootstrap можна використовувати для відображення стану завантаження у ваших проектах. Вони побудовані лише з використанням HTML та CSS, тому вам не потрібний JavaScript. Вам, однак, знадобиться використовувати деякі параметри JavaScript для перемикання їх відображення. Зовнішній вигляд, вирівнювання та розміри можна легко налаштувати за допомогою наших службових класів.

З метою доступності кожен завантажувач містить role="status" і вкладений <span class="sr-only">Loading...</span>.

Ефект анімації цього компонента залежить від медіазапиту prefers-reduced-motion.

Спливаючі повідомлення

Налаштовуйте спливаючі повідомлення для попередження відвідувачів.

Спливаючі повідомлення (Toasts) - це легкі сповіщення, розроблені для імітації push-повідомлень, які були популярні в мобільних та настільних операційних системах. Вони побудовані з flexbox, тому їх легко вирівняти та розташувати.

Огляд

Що потрібно знати під час використання:

  • Спливаючі повідомлення включені з міркувань продуктивності, тому ви повинні ініціалізувати їх самостійно.
  • Спливаючі повідомлення автоматично ховаються, якщо ви не вкажете autohide: false.

Ефект анімації цього компонента залежить від медіазапиту prefers-reduced-motion.

Приклад

Основний

Щоб заохотити розтяжні та передбачувані спливаючі повідомлення, ми рекомендуємо поміщати їх у заголовок та тіло. У заголовках Спливаючі повідомлення використовується display: flex, що дозволяє легко вирівнювати вміст завдяки нашим утилітам margin та flexbox.

Спливаючі повідомлення можна гнучко налаштовувати під ваші потреби з мінімальними настройками. Як мінімум необхідно, щоб один «елемент» містив ваш контент і настійно рекомендуємо використовувати кнопку закриття.
